# Java Collection Framework (Java 8)

## Definition
The **Collection Framework** in Java is a unified architecture that provides a set of **interfaces** and their **implementations** to store, retrieve, and manipulate groups of objects (collections).

---

## Key Points
- Defines **standard interfaces** (like List, Set, Map, Queue)
- Provides **ready-to-use implementations** (like ArrayList, HashSet, HashMap)
- Helps in **efficient data handling and manipulation**
- Reduces programming effort by providing built-in data structures

---

## Core Interfaces
- `List` → Ordered collection (allows duplicates)
- `Set` → Unordered collection (no duplicates)
- `Map` → Key-value pair structure
- `Queue` → Used for processing elements in order

---

## Common Implementations
| Interface | Implementation |
|----------|--------------|
| List     | ArrayList, LinkedList |
| Set      | HashSet, TreeSet |
| Map      | HashMap, TreeMap |
| Queue    | PriorityQueue |
